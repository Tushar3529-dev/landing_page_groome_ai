# Groome

Production-oriented Flutter Web landing site for Groome, a salon booking and
business management platform.

## Architecture

The app follows feature-first Clean Architecture:

- `presentation`: pages, reusable widgets, and `ContactCubit`
- `domain`: `ContactLead`, repository contract, and submit use case
- `data`: Firestore datasource and repository implementation
- `core`: theme, responsive helpers, shared widgets, constants, and services

Dependencies are registered with `get_it`, navigation uses `go_router`, and
presentation state uses `flutter_bloc`.

## Firebase setup

1. Create a Firebase project with Firestore, Hosting, and Cloud Functions.
2. The included `.firebaserc` targets `groom-ai-d91fe`. Update it if you move
   this app to another Firebase project.
3. Add the Gmail App Password for `support@groome.net` as a Secret Manager
   secret:

   ```powershell
   firebase functions:secrets:set GMAIL_APP_PASSWORD
   ```

   Use a Gmail App Password for `support@groome.net`, never the account
   password.

4. Resolve packages:

   ```powershell
   flutter pub get
   Set-Location functions
   npm install
   Set-Location ..
   ```

## Run locally

Firebase configuration is read from `lib/firebase_options.dart`, generated by
FlutterFire for the configured web app. Run the site with:

```powershell
flutter run -d chrome
```

Optional staging/override builds can still supply Firebase configuration through
dart-defines:

```powershell
flutter run -d chrome `
  --dart-define=FIREBASE_API_KEY=... `
  --dart-define=FIREBASE_APP_ID=... `
  --dart-define=FIREBASE_MESSAGING_SENDER_ID=... `
  --dart-define=FIREBASE_PROJECT_ID=... `
  --dart-define=FIREBASE_AUTH_DOMAIN=... `
  --dart-define=FIREBASE_STORAGE_BUCKET=...
```

If dart-defines are supplied, they take precedence over the generated web
options.

## Contact email test

The automated Functions test verifies that a submitted contact lead is converted
into the Gmail notification sent from `support@groome.net` to
`Maanyam@groome.net`.

The email body is:

```text
The new user needs to contact you.

Name:
...

Email:
...

Phone no:
...

Business name:
...

Message:
...

Groome
```

For a real inbox test, deploy the site and Functions with `GMAIL_APP_PASSWORD`
configured for `support@groome.net`, then submit the Contact form with a test
lead. Confirm that `Maanyam@groome.net` receives the "New user contact request"
email from `support@groome.net` and that replying targets the lead email
address.

If the Contact form shows success but no email arrives, check that the latest
Function has been deployed and then inspect the function logs:

```powershell
firebase deploy --only functions
firebase functions:log --only notifyOwnerOfNewLead
```

## Deploy

```powershell
firebase deploy
```

This deploys Hosting, Firestore rules, and the lead notification function.

## SEO notes

The web shell includes title, description, Open Graph, Twitter Card, canonical,
and SoftwareApplication structured-data tags. Update the `groome.app` URLs in
`web/index.html` if the production domain differs. Flutter renders the app
client-side, so a content-heavy marketing strategy may also benefit from
prerendered route snapshots at the CDN layer.
